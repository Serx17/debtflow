# Расширение портфолио DebtFlow: Scorer и Compliance

План развития портфолио на базе единой платформы DebtFlow: добавление модулей Pre-Collection Scorer (ML-скоринг) и Compliance & 230‑ФЗ в один репозиторий и один деплой.

---

## 1. Рекомендация по архитектуре

**Реализовать Scorer и Compliance как модули/разделы внутри DebtFlow: один репозиторий, один деплой.**

- Один живой продукт с полным циклом: скоринг → кампании → compliance и отчётность.
- Переиспользование данных: `debtors`, `communication_log`, общий Supabase и UI.
- Для работодателя: одна ссылка, один вход — дашборд, шаблоны, кампании, Scorer, Compliance.

---

## 2. Структура продукта после расширения

### Маршруты и навигация

| Раздел | Маршрут | Назначение |
|--------|--------|------------|
| Дашборд | `/dashboard` | Метрики, графики, недавние коммуникации (как сейчас) |
| Должники | `/debtors` | Список, импорт CSV, фильтры (как сейчас) |
| Шаблоны | `/templates` | SMS / Email / Call (как сейчас) |
| Каналы | `/channels` | Настройки mock-каналов (как сейчас) |
| Коммуникации | `/communications/new`, `/communications` | Мастер кампаний и лог (как сейчас) |
| **Scorer** | **`/scorer`** | Pre-Collection: скоринг должников (ML), сегменты, рекомендация канала |
| **Compliance** | **`/compliance`** | Лимиты 230‑ФЗ по контактам, время, отчёт для ФССП |
| Юридические | `/legal/*` | Политика, условия (как сейчас) |
| Админ | `/admin/seed` | Демо-данные (как сейчас) |

В навигации (layout/sidebar) добавить пункты: **Scorer** и **Compliance**.

### Общие данные (текущая схема)

- **organizations**, **users** — без изменений.
- **debtors** — используется в Scorer (фичи для модели: debt_amount, region; при необходимости добавить поля `days_overdue`, `first_overdue_date` миграцией).
- **communication_log** — используется в Compliance (подсчёт контактов по debtor_id, channel, sent_at для проверки лимитов 230‑ФЗ и формирования отчёта).
- **templates**, **channel_settings** — без изменений для новых модулей (Scorer может рекомендовать канал по сегменту; Compliance не меняет шаблоны).

При необходимости:
- Таблица **debtor_scores** (debtor_id, score, segment, model_version, computed_at) для хранения результатов скоринга.
- Константы или справочник лимитов 230‑ФЗ (звонки: 1/2/8, SMS: 2/4/16, окна 8–22 / 9–20) в коде или в БД.

## 3. Модуль Pre-Collection Scorer (`/scorer`)

**Цель:** сегментация должников по вероятности возврата, рекомендация канала/стратегии, демонстрация ML в продукте.

**Функциональность MVP:**
- Загрузка/выбор портфеля из существующих `debtors` (по organization_id).
- Расчёт скора: **модель машинного обучения** (логистическая регрессия, дерево или лёгкий бустинг) по фичам: debt_amount, region, при наличии — days_overdue / история из communication_log. Обучение на демо-данных (synthetic или помеченные по status sent/failed).
- Сегменты: например «мягкое напоминание», «активная работа», «судебное взыскание» по порогам скора.
- UI: таблица должников с колонками скор, сегмент, рекомендуемый канал; фильтры по сегменту; опционально — экспорт сегмента для использования в кампаниях (ссылка на `/communications/new` с предзаполненным списком должников).
- Технически: API route в Next.js, вызывающий скрипт/сервис на Python (например, через child_process или отдельный HTTP API) с обученной моделью, либо реализация простой модели на TypeScript/JS (если без Python). Модель переобучается по кнопке или по расписанию на демо-данных.

**ML:** в рамках MVP явно заложить обучение и предсказание (даже на синтетике), чтобы показать навыки data science и интеграцию модели в продукт.

## 4. Модуль Compliance & 230‑ФЗ (`/compliance`)

**Цель:** контроль соблюдения лимитов контактов 230‑ФЗ и подготовка данных для отчётности ФССП (с сентября 2026).

**Функциональность MVP:**
- Агрегация по `communication_log`: по каждому должнику (debtor_id) за выбранный период — число звонков (channel = 'call'), SMS (channel = 'sms') по дням, неделям, месяцам; проверка времени отправки (sent_at) на попадание в разрешённые окна (будни 8:00–22:00, выходные 9:00–20:00).
- Справочник лимитов 230‑ФЗ в коде: звонки 1/2/8, SMS 2/4/16 (в день/неделю/месяц).
- UI: экран «Соответствие 230‑ФЗ» — список должников с индикатором (соблюдено / превышение по каналу или по времени); детализация по должнику (сколько контактов за период). Кнопка «Экспорт для отчётности» — выгрузка CSV/Excel с полями, релевантными отчёту для ФССП (организация, период, должник, каналы, количество, факты вне окон при наличии).
- ML в Compliance не используется; при необходимости в будущем можно использовать сегменты из Scorer для приоритизации проверки или отчёта.

---

## 5. Рынок взыскания 2025–2026 (кратко)

- Тройное давление: рост банкротств и просрочки, ужесточение 230‑ФЗ и отчётность ФССП с 09.2026, рост исполнительского сбора.
- Взыскивается 3–7% от суммы долга; тренд на pre-collection и скоринг (удержание до 97% на ранних стадиях при автоматизации).
- Боли работодателя: соответствие 230‑ФЗ и подготовка к ФССП; рост возвратов за счёт сегментации и приоритизации; прозрачность и аудит контактов.

## 6. Итоговая картина портфолио

| Блок | Боль работодателя | Экспертиза |
|------|-------------------|------------|
| DebtFlow (текущий) | Омниканальный цикл, лог, метрики | Юрист + опер + тех |
| Scorer | Pre-collection, рост возвратов, ML | Аналитика + ML + продукт |
| Compliance | 230‑ФЗ, отчётность ФССП, штрафы | Юрист + продукт + данные |

Один репозиторий, один деплой, одна ссылка для демо — платформа взыскания с аналитикой и регуляторикой.

## 7. Порядок реализации (предложение)

1. **Compliance** — агрегации по `communication_log`, лимиты в коде, страница `/compliance`, экспорт. Без новых таблиц или с минимальными (например, кэш агрегатов по периодам при росте данных).
2. **Scorer** — таблица `debtor_scores` (при необходимости), фичи из `debtors` (+ миграция полей при необходимости), обучение модели на демо-данных, API предсказания, страница `/scorer` с таблицей и сегментами, рекомендация канала.
3. Интеграция в навигацию и README: описание разделов Scorer и Compliance, обновление демо-сценария (скоринг → кампании → проверка compliance и отчёт).
