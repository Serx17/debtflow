## DebtFlow – MVP для портфолио

DebtFlow — это прототип омниканального конструктора коммуникаций для коллекторских агентств и банков. Цель продукта — показать, как можно автоматизировать работу с должниками: загружать списки, настраивать шаблоны сообщений, имитировать отправку по каналам (SMS, email, звонки) и анализировать результаты.

Проект создаётся как демонстрация личной экспертизы на стыке трёх ролей:
- юрист по взысканию задолженности;
- операционный директор в компании по лидогенерации для банков;
- технический специалист (голосовые боты, SQL/аналитика, Python, API).

### Стек и инфраструктура

- Next.js (App Router, TypeScript) — фронтенд и серверные обработчики.
- Supabase — база данных, авторизация и хранение.
- Tailwind CSS + shadcn/ui — дизайн-система и UI-компоненты.
- Vercel — деплой и хостинг демо-версии.
- GitHub — репозиторий и код для портфолио.

### Ключевые пользовательские сценарии MVP

1. Загрузка списка должников (синтетические данные).
2. Просмотр и фильтрация должников в удобной таблице.
3. Создание шаблонов сообщений для SMS, email и звонков (Call).
4. Настройка \"каналов\" (моки) с указанием стоимости сообщений.
5. Запуск коммуникаций по выбранным должникам с выбранным шаблоном.
6. Имитация отправки (mock-сервис) с логированием статусов.
7. Дашборд отчётности: объёмы отправок, успешность, стоимость.

### Этапы разработки (оглавление)

1. Организация работы и фиксированный мастер-план.
2. Проверка и базовая настройка проекта (Next.js + Tailwind + shadcn/ui).
3. Настройка Supabase и схемы БД.
4. Авторизация через Supabase Auth.
5. Страница `Debtors` (список и импорт CSV).
6. Страница `Templates` (шаблоны SMS/Email).
7. Страница `Channels` (настройки mock-каналов).
8. Мастер-коммуникаций `/communications/new`.
9. Логи коммуникаций и дашборд отчётности.
10. Демо-данные и страница `/admin/seed`.
11. Безопасность и RLS в Supabase.
12. Деплой на Vercel и связка с GitHub.
13. Презентационная упаковка (README, юридические заглушки, демо-видео).

По мере разработки сюда можно добавлять разделы:
- заметки по архитектуре (как устроены сущности и связи);
- технические решения (например, как реализован mock-отправитель);
- идеи для будущих версий (planned features).
